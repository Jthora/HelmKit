---

## 1. Goal

Close the gap between *firmware-on-master* and *operator-can-bench-
without-friction*, and pre-stage the hardware-gated Track J commits so
sensor arrivals translate into single-afternoon landings rather than
multi-day driver-design sessions.

Two halves:

- **Bench-side**: ship the operator's feedback loop. Capture works,
  but there is no tool that says *"did the session look sane?"* until
  Track J commit 7 (`analyze_g2.py`), which is gated on an AD8232 we
  don't have yet. That's the wrong gate for PPG/temp/GSR sanity.
- **Latency-side**: pre-stage J-4 (MAX30205), J-5 (AD8232), J-6
  (ECG-band Pan-Tompkins) so the hardware-arrival commit is "drop in
  the library call + smoke gates" instead of "design the driver."

---

## 2. Scope

In scope:

- New tool `tools/analyze_capture.py`: channel-agnostic NDJSON
  inspector. Per-channel: sample count, observed rate vs SCHEMA-
  expected rate, in-range percentage, `q` flag distribution, gap
  detection, mean/std for numeric channels. One-page report. No
  oracle required.
- New tool unit tests + fixture NDJSON under `tests/fixtures/` —
  enough that the analyser is validated offline in CI without a
  Heltec attached.
- New doc `docs/firmware/mk0.5_bench_quickstart.md`: single-page
  wire → flash → capture → inspect runbook covering Bridges A + B.
  Consolidates the relevant slices of `mk0.5_wiring.md`,
  `capture_ndjson.py` README, `SCHEMA.md`, and Track J §4 so the
  operator reads one doc, not four.
- Firmware scaffolds for J-4 / J-5 / J-6 that do **not** require
  hardware to compile and lint:
  - `firmware/mk0.5/src/drivers/max30205.cpp` skeleton matching the
    MLX90614 pattern (Adafruit lib hidden behind `void* impl_`,
    floor-enforced period, sticky-Health rules, smoke test gated on
    `kNotImplemented` until hardware lands).
  - `firmware/mk0.5/src/drivers/ad8232.{h,cpp}` skeleton: ADC1 read
    path on `kAd8232Out=5`, lead-off watch on GPIO 6/7, smoke gated
    on `kNotImplemented`. Header-shape only.
  - `firmware/mk0.5/src/dsp/r_peak.h`/.cpp refactor to lift band
    coefficients into a `RPeakBand` config struct: PPG (current
    defaults) and ECG (5–15 Hz bandpass, 150 ms MWI window). No
    behaviour change for PPG; ECG path callable but unused until J-6.

Out of scope (defer):

- The actual hardware-validated J-4 / J-5 / J-6 landings — those are
  Track J commits, gated on MAX30205 (~May 27) and AD8232 (~Jun 1–15).
- `analyze_g2.py` — remains Track J commit 7. K-1's analyser is the
  general-purpose substrate; G2 closure stays in Track J where it
  belongs.
- Any change to canonical NDJSON wire format or pin assignments.

## 4. Acceptance gates per commit

**K-1** — `analyze_capture.py`:

- Runs against `captures/*.ndjson` with no arguments beyond a path.
- Per channel reports: count, duration_s, observed_hz, expected_hz
  (looked up from a small table mirroring SCHEMA §2.2), rate_ratio,
  in_range_pct, q_distribution (counts per `q` value), gap_count
  (defined: any inter-sample dt > 2× expected period), value mean/std
  for numeric `v` fields, channels-seen vs channels-expected.
- Exit code 0 on success; non-zero only on file/parse error (does not
  fail on "session looked bad" — that is the report's job).
- Importable as a library so K-7 (Track J `analyze_g2.py`) can call
  into it later.

**K-2** — fixtures + tests:

- Three fixtures: `ppg_60s_good.ndjson`, `mlx_gsr_30s_good.ndjson`,
  `ppg_60s_with_gap.ndjson` (synthetic; hand-rolled or generated by a
  tiny script).
- Tests cover: rate detection within 5 % of expected; gap detection;
  in-range percentage; q-distribution counting; channels-expected
  table.
- `pytest tests/test_analyze_capture.py` runs clean.

**K-3** — `mk0.5_bench_quickstart.md`:

- One page (roughly), reads top-to-bottom: wire, flash, capture,
  inspect, expected output. Bridge A (PPG-only) is the headline path;
  Bridge B (MLX + GSR) is an addendum.
- Cross-links into `mk0.5_wiring.md`, `SCHEMA.md`, `capture_ndjson.py`,
  `analyze_capture.py` — but does not duplicate them.
- Lint clean via `tools/check_links.py`.

**K-4** — MAX30205 scaffold:

- Compiles. `mk0.5` firmware build still green.
- `max30205_smoke_test()` returns `SmokeFail::kNotImplemented` until
  hardware lands; this is intentional and documented in-source.
- File shape matches `mlx90614.cpp` (void* impl_, period floor,
  health enum, Wire1 @ 0x48/0x49 placeholder).
- `Max30205Sample` struct stable so K-7 can wire emit code without
  another refactor.

**K-5** — AD8232 scaffold:

- Compiles. Pins documented in `pins.h` (already present).
- Lead-off watch header declares the API surface even if the
  implementation is a stub.
- `ad8232_smoke_test()` returns `SmokeFail::kNotImplemented`.

**K-6** — `RPeakBand` config:

- PPG path produces byte-identical RR output vs current `master` on
  the PPG fixture from K-2 (regression-tested).
- ECG band coefficients are present, callable, and dead-code-clean
  (no path activates them until J-6 lands).
- No NDJSON wire change.

## 5. Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| `analyze_capture.py` grows into `analyze_g2.py` scope creep | medium | Hard rule: no oracle, no HRV math, no pass/fail gates in K-1. Those are J-7. K-1 is a *reporter*. |
| Pre-staged scaffolds drift before the hardware lands (lib API changes, SCHEMA tweaks) | low | Scaffolds live in firmware tree under normal lint/build; CI will catch drift. |
| `RPeakBand` refactor introduces a PPG regression | medium | K-2 fixture includes a recorded PPG session; K-6 acceptance gate is byte-identical RR output against it. |
| Bench quickstart drifts from `mk0.5_wiring.md` | low | Quickstart cross-links rather than restates. Wiring authority remains in `mk0.5_wiring.md`. |

## 7. Relationship to Track J

Track K is **not** a replacement or fork of Track J. It is the
preparation layer underneath J-4 / J-5 / J-6 / J-7:

- K-1 (analyser substrate) → J-7 (`analyze_g2.py`) imports it.
- K-4 (MAX30205 scaffold) → J-4 fills it in.
- K-5 (AD8232 scaffold) → J-5 fills it in.
- K-6 (`RPeakBand`) → J-6 selects the ECG band.

When Track K lands, Track J §5 commits 4 / 5 / 6 / 7 each shrink to
"add the library call, smoke gates, and emit code" with no
architectural decisions left to make.
